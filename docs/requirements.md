# 1. はじめに

## 1.1 目的

本書は、戦国時代の日本史を地理情報と年代情報の両軸から探索できるWebサイト「戦国歴史マップ」のMVP要件を定義することを目的とする。

本システムは、従来の年表型・文章型の歴史学習では把握しづらい「地域ごとの同時代性」「出来事の地理的関係」「勢力や事件の空間的広がり」を、地図・年代スライダー・イベント吹き出しによって直感的に理解できるようにする。

---

## 1.2 適用範囲

本書の対象範囲は、以下のMVP機能である。

* 戦国時代の主要出来事の表示
* 年代スライダーによる出来事の絞り込み
* 日本地図上への出来事ピン・吹き出し表示
* 全国表示・地域表示の切り替え
* 出来事詳細カードの表示
* PC・スマートフォン向けレスポンシブ表示
* 出典情報の表示

以下はMVP対象外とする。

* ユーザー登録
* 投稿機能
* コメント機能
* 管理画面
* 課金機能
* AIによるリアルタイム文章生成
* 完全な旧国境界・勢力圏の時系列再現
* 全時代対応
* 多言語対応

---

## 1.3 用語定義

| 用語      | 定義                              |
| ------- | ------------------------------- |
| 戦国時代    | 本MVPでは原則として1467年から1615年までを対象とする |
| 出来事     | 合戦、政変、同盟、滅亡、築城、文化的事件など、歴史上のイベント |
| イベントカード | 出来事の概要・年・地域・関係人物・出典を表示するUI部品    |
| 吹き出し    | 地図上の地点に紐づいて表示される短いイベント概要        |
| ピン      | 地図上の出来事発生地点を示すマーカー              |
| 年代スライダー | 表示対象の年代を選択するUI部品                |
| 地域      | MVPでは地方ブロックまたは都道府県相当の区分を指す      |
| 旧国      | 尾張、三河、美濃、甲斐、越後、山城など、近世以前の令制国    |
| 重要度     | イベントを地図上に優先表示するためのスコア           |
| 出典      | イベント情報の根拠となる資料・Webページ・書籍・データベース |

# 2. システム概要

## 2.1 システムコンセプト

本システムは、戦国時代の歴史出来事を、地図と年代操作によって探索できるWebサイトである。

ユーザーは、年代スライダーを操作することで、特定年または年代範囲に発生した出来事を日本地図上で確認できる。地図上にはイベントピンや吹き出しが表示され、クリックまたはタップすることで詳細情報を確認できる。

---

## 2.2 提供価値

本システムは、以下の価値を提供する。

| 価値    | 内容                                   |
| ----- | ------------------------------------ |
| 地理的理解 | どこで何が起きたかを地図上で把握できる                  |
| 同時代理解 | 同じ年代に複数地域で起きた出来事を比較できる               |
| 学習補助  | 日本史学習における年表暗記を地理理解で補完する              |
| 探索性   | 興味のある地域・年代から歴史を掘り下げられる               |
| 視覚的魅力 | クラシックかつモダンな地図UIにより、歴史サイトとしての体験価値を高める |

---

## 2.3 想定ユーザー

| ユーザー種別    | ニーズ                  |
| --------- | -------------------- |
| 中高生       | 戦国時代の出来事を地理と年表で理解したい |
| 歴史好き      | 地域別・年代別に戦国史を探索したい    |
| 旅行者       | 旅行先に関係する歴史出来事を知りたい   |
| 創作・ゲーム制作者 | 時代背景や地域情勢を調べたい       |
| 一般ユーザー    | 地図を眺めながら気軽に戦国史を楽しみたい |

---

# 3. 利用シナリオ

## 3.1 全国の出来事を見る

ユーザーは年代スライダーで「1560年」を選択する。
システムは、1560年前後に日本全国で発生した主要イベントを地図上に表示する。
重要イベントである「桶狭間の戦い」は吹き出しとして表示される。その他の関連イベントはピンとして表示される。

---

## 3.2 地域ごとの出来事を見る

ユーザーは地域選択で「東海」を選択する。
システムは、選択中の年代に東海地方で発生した出来事を優先表示する。
地図上では東海地方がハイライトされ、関連イベントが表示される。

---

## 3.3 出来事の詳細を見る

ユーザーは地図上の「桶狭間の戦い」の吹き出しをクリックする。
システムはイベント詳細カードを表示する。
カードには、年、地域、概要、関係人物、カテゴリ、出典が表示される。

---

## 3.4 スマートフォンで閲覧する

ユーザーはスマートフォンでサイトにアクセスする。
システムは、地図操作を主導線にせず、年代選択とイベントカード一覧を中心に表示する。
地図上のピンをタップすると、画面下部にボトムシート形式でイベント概要が表示される。

---

# 7. データ要求

## 7.1 イベントデータ

イベントデータは、以下の項目を持つ。

| 項目                  | 必須 | 内容             |
| ------------------- | -- | -------------- |
| id                  | 必須 | イベントの一意ID      |
| title               | 必須 | イベント名          |
| year_start          | 必須 | 開始年            |
| year_end            | 任意 | 終了年            |
| date_text           | 任意 | 月日・旧暦などの表示用文字列 |
| period              | 必須 | 時代区分           |
| categories          | 必須 | カテゴリ           |
| regions             | 必須 | 地域             |
| old_provinces       | 任意 | 旧国名            |
| latitude            | 任意 | 緯度             |
| longitude           | 任意 | 経度             |
| summary_short       | 必須 | 吹き出し用短文        |
| summary_long        | 任意 | 詳細説明           |
| persons             | 任意 | 関係人物           |
| factions            | 任意 | 関係勢力           |
| importance          | 必須 | 表示優先度          |
| sources             | 必須 | 出典             |
| verification_status | 必須 | 確認状態           |

---

```json
  "persons": ["織田信長", "今川義元", "松平元康"],
  "factions": ["織田氏", "今川氏"],
  "importance": 5,
  "sources": [
    {
      "name": "出典名",
      "url": "https://example.com",
      "checked_at": "2026-07-08",
      "license_note": "要確認"
    }
  ],
  "verification_status": "verified"
}
```

## 7.3 重要度定義

| importance | 意味     | 表示方針         |
| ---------: | ------ | ------------ |
|          5 | 最重要    | 初期表示で吹き出し候補  |
|          4 | 重要     | 地図ラベルまたは優先ピン |
|          3 | 通常     | ピン表示         |
|          2 | 補足     | 一覧中心         |
|          1 | 詳細資料向け | 検索時のみ表示      |

---

## 7.4 確認状態

| status    | 意味     |
| --------- | ------ |
| draft     | 下書き    |
| reviewing | 確認中    |
| verified  | 出典確認済み |
| rejected  | 採用見送り  |

本番表示対象は `verified` のみとする。

# 18. 判断

本システムは、MVPとして「戦国時代 × 日本地図 × 年代スライダー × 吹き出し表示」に絞ることで、十分に成立する。

特に、戦国時代は地域性が強く、合戦・大名・城・政変が地理情報と結びつきやすいため、本システムの初期対象として適している。

一方で、史実データの正確性、出典管理、スマートフォンUI、地図上の情報過多は主要リスクである。
そのため、MVPでは「重要イベントのみ吹き出し表示」「その他はピン表示」「スマホはカード中心」という方針を採用する。

以上をもって、本MVPの初期要件とする。
